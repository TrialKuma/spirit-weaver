## 核心系统架构

### 1.1 CTB 时间轴系统 (Conditional Turn-Based)

**核心公式**：
- **行动值 (AV)**：`AV = 10000 / Speed`
- **跑条机制**：所有单位在同一条轨道上跑条，AV 归零者获得行动权
- **行动消耗**：标准行动后，重置 `AV = 10000 / Speed`
- **暂停机制**：当有单位到达行动条底部（AV归零）时，所有单位的AV停止减少，直到该单位行动完成

**v3.0 - 节奏优化（动态跑条速度）**：
- 当下一个行动单位与当前单位**同属一方**时（连续玩家回合），跑条动画加速至 **3-5 倍速**
- 当下一个行动单位为**对方**时（玩家→敌人交替），恢复正常速度，自然制造节奏停顿
- **目的**：消除连续行动之间的等待感，同时保留速度差带来的"多动"体验
- **视觉表达**：快进时时间轴上的单位标记显示加速残影效果，强化"我真的很快"的感受

**实现细节**：
- 每 tick 所有单位 AV -1
- 当 AV ≤ 0 时，该单位获得行动权
- 行动完成后重置 AV，解除暂停状态
- 敌人攻击时也会暂停行动条，等待表现播放完成（1.5秒）

### 1.2 伤害计算系统

**基础公式**：
- `Damage = (BaseAtk * SkillMultiplier) * BuffModifiers`
- 多hit技能：每发独立计算防御
- 最终伤害：`FinalDamage = max(1, DamagePerHit - EnemyDef)`

**防御机制**：
- 每发伤害都会减去敌人防御值
- 最小伤害为 1 点

### 1.3 Buff系统

**层数制**：如[蓄力]（层数消耗）  
**回合制**：如[易伤]（持续回合）  
**极性制**：如[平衡状态]（正负值影响攻防）

## 单局流程结构（v3.0 新增）

### 2.1 整体结构

**节奏模型**：(3+1) × 3 + 1

```
阶段1: 普通战 → 普通战 → 精英战 → 修整
阶段2: 普通战 → 普通战 → 精英战 → 修整
阶段3: 普通战 → 普通战 → 精英战 → 修整
最终:  BOSS战
```

**节点总计**：9场战斗 + 3次修整 + 1场最终BOSS = 13个节点

### 2.2 单局完整流程

```
选择职业 → 魂灵祭坛（选魂灵）→ 阶段1 → 阶段2 → 阶段3 → 最终BOSS → 通关/失败
```

### 2.3 战斗节点

**普通战**：
- 从当前阶段的敌人池中随机匹配
- 战斗结束后获得一次强化（三选一，标准稀有度池）

**精英战**（每阶段第3场）：
- 拥有独特机制的强敌（不仅是数值膨胀）
- 战斗结束后获得一次强化（三选一，**高稀有度池**：SR/SSR 概率提升）

**最终BOSS**：
- 固定BOSS，拥有多阶段或特殊机制
- 通关即胜利

### 2.4 修整节点

每阶段战斗结束后进入修整，**随机出现以下内容**：

- **商店**：消耗金币购买强化/道具/回血
- **回血**：恢复一定比例生命值
- **事件**：随机事件，可能有风险-收益的选择（如"献祭20%生命换一个SSR强化"）

### 2.5 难度递增

- **阶段间**：每阶段敌人基础属性提升（HP/ATK/DEF 按比例增长）
- **阶段内**：第3场精英战比前两场普通战强，精英有独特行为模式
- **最终BOSS**：独立设计，不套用普通敌人模板

### 3.5 判官 (Balance Master) - 极值决策型

**资源系统**：
- **资源名称**：平衡 (Balance)
- **范围**：-5 到 +5
- **初始值**：0
- **颜色**：紫色 (#9933ff)

**资源回转逻辑**：
- **自动归中**：每回合开始时，数值向 0 靠拢 1 点
- **操作逻辑**：阳击向正偏移 +3，阴击向负偏移 -3
- **极值状态**：平衡达到 ±5 时触发极值（阳极/阴极），下回合激活

**被动技能 — 阴阳之道**：
- 阳面（> 0）：自身全属性 +10%/点（ATK、DEF、Speed）
- 阴面（< 0）：敌方攻防 -10%/点（ATK、DEF）

**反差机制**：
- 逆方向使用技能时，额外获得 |当前平衡值| × 15% 的倍率加成
- 顺方向无反差加成

**主动技能**：

| 技能ID | 技能名 | 类型 | 倍率 | 平衡偏移 | 附加效果 | 极值增强 |
|--------|--------|------|------|----------|----------|----------|
| yang_strike | 阳击 | 攻击 | 130% | +3 | 行动值推进15% | 烈阳击：250%，无视全部DEF |
| yin_strike | 阴击 | 特技 | 100% | -3 | 侵蚀（ATK×20%回合伤，1回合） | 深渊击：180%，深渊侵蚀（ATK×40%，2回合） |
| verdict | 宣判 | 能量技 | 动态 | →0 | 仅极值状态可用，平衡强制归零 | 阳极：(100+80×落差)%=500%满额；阴极：(60+50×落差)%=310%满额+审判(受伤+40%,2回合) |

**极值系统**：
- **触发**：技能结算后 |balance| = 5，标记极值，下回合开始时激活
- **持续**：1 回合（暂停归中，维持 ±5）
- **极值回合三选一**：
  - 同方向（极值增强）：强力攻击，消耗极值 buff，平衡维持 ±5
  - 逆方向（反差）：反差满额加成 + 控制效果，消耗极值 buff，平衡维持 ±5
  - 宣判（归零）：落差爆发，消耗极值 buff，平衡→0（伤害在归零后结算，不享受被动加成）
- **CD**：极值消耗后 3 次行动内不可再次触发

**节奏**：
- 从 0 推到极值：2 回合
- 极值→同方向→下次极值：4 回合（CD 期间维持高位被动）
- 极值→宣判→下次极值：4 回合（CD 期间从 0 重新推条，可换方向）

**核心体验**：纯决策翻条。每回合都在权衡"继续投资"vs"反差兑现"，每次极值都在纠结"维持高位"vs"归零爆发"。

## 追加技能系统（v3.0 新增）

### 4.1 核心机制

**触发方式**：概率触发 + 手动确认
- 使用主动技能后，根据该技能配置的追加触发率，概率判定是否触发追加选项
- 触发率范围：**30%-50%**，按技能和职业不同调整
- 触发后弹出追加技能面板，**玩家手动确认**是否使用

**资源消耗**：
- 追加技能使用需要消耗**少量职业资源**（1-2点）
- **跳过追加**：不消耗资源，回合正常结束

**设计意图**：
- **触发时**：正反馈感（"触发了！"）
- **决策点**：消耗资源使用 vs 保留资源跳过——取决于当前资源规划和战场局势
- **没触发时**：按原计划执行，不影响基础循环
- **触发了**：可能优化你的计划（追加伤害/防御），也可能打乱你的资源规划（消耗了本来要攒的资源）

### 4.2 追加技能分类

每个职业拥有 **2-3 个**追加技能，大致分为：

| 类型 | 作用 | 举例方向 |
|------|------|---------|
| 攻击追加 | 额外输出/追击 | 追加一次打击、追加DOT、追加减防 |
| 防御追加 | 护盾/减伤/回血 | 获得小护盾、短暂减伤 |
| 功能追加 | 控制/减益/资源 | 概率控制1回合、回复少量资源 |

### 4.3 触发规则

- 不同主技能可触发**不同的追加技能**
- 同一主技能若绑定多个追加，触发时**随机选定一个**弹出
- **追加不连锁**：追加技能不会再触发追加
- 追加执行**不消耗行动**（不重置AV），属于当前行动的延伸

### 4.4 UI表现

- **触发提示**：主技能效果结算后，追加技能卡片在技能面板亮起（闪光/高亮）
- **信息显示**：追加技能名称、效果描述、资源消耗
- **操作按钮**："使用"和"跳过"
- **无时间限制**：等待玩家操作，不自动消失

## 魂灵系统设计

### 5.1 选择流程（v3.0 更新）

**时机**：选择职业后、第一场战斗前

**流程**：
1. 进入**魂灵祭坛**独立场景
2. 随机决定一个魂灵大类（自律/协力/助理）
3. 该大类内随机展示 3 个魂灵，玩家 3 选 1
4. 播放契约动画
5. 进入阶段1第一场战斗

**表现设计**（哈迪斯式祝福）：
- 全屏祭坛场景，暗色背景，中央符文阵/祭坛
- 三个魂灵以光球/符文/幻影形态悬浮于祭坛上方
- 悬停显示魂灵详细信息（名称、类型、效果描述）
- 选中后其他两个消散，选中的飞向玩家位置（环绕在身边）
- CSS过渡动画实现，无需额外美术资源

### 5.2 时间轴规则

- 自律型魂灵走 CTB 行动条
- 商人为特例：虽属助理型，但也走 CTB 行动条

### 5.3 自律型魂灵（走条）

- **守卫**（SPD 140）：行动后提供 **100% 攻击力护盾**；若上一次技能消耗 ≥6 资源，护盾将**反射等量伤害**
- **誓盟**（SPD 140）：行动时造成 **100% 攻击力伤害**；玩家每次命中使其行动**提前 20%**
- **领主**（SPD 120）：行动时造成 **150% 攻击力伤害**；每消耗 1 点资源，本次伤害 **+10%**

### 5.4 协力型魂灵（不走条）

- **伴侣**：玩家每次攻击后追加 **80% 攻击力伤害**
- **侍从**：攻击/特技追加 **60%**；能量技追加 **120%**
- **协奏**：攻击/特技叠【蓄势】；能量技追加 **120% × (1 + 25% × 层数)** 并清空

### 5.5 助理型魂灵（不走条，商人除外）

- **书记**：攻击/特技叠【记录】；能量技前每层 **+10% 伤害**并回 **10% 资源**
- **信使**：发布行动任务，完成后获得随机奖励并刷新任务
- **商人**（SPD 200，走条）：行动时获得 **1 金币**；资源满溢额外得币；能量技消耗金币回资源（**每枚 10%**）

### 5.6 特殊规则

**平衡系资源处理**：
- 助理型导致资源变化时，若资源为**平衡**，则按当前方向**继续向外推**
- 平衡值为 **0** 时不发生方向推动
- 变化发生在造成伤害之前，且**不允许超出上限**

**信使任务节奏**：
- 任务完成后立即发放奖励，**奖励持续到下一次行动结束**
- **隔回合发布**机制：完成后下一回合不发布新任务

**肉鸽扩展接口**：
- 所有倍率与阈值以 `tunables` 配置保存，便于强化时叠加/替换

## 战斗机制

### 7.1 回合流程（v3.0 更新）

1. **行动条跑条**：所有单位 AV 持续减少（同侧连续时加速动画）
2. **单位就绪**：当单位 AV ≤ 0 时，暂停所有单位，该单位获得行动权
3. **执行行动**：
   - 玩家：选择技能使用
   - 敌人：执行意图行动
4. **追加判定**（v3.0 新增）：
   - 概率检查是否触发追加技能
   - 若触发：弹出追加面板，等待玩家确认
     - 使用 → 消耗资源，执行追加效果
     - 跳过 → 不消耗，进入下一步
   - 若未触发：直接进入下一步
5. **表现播放**：伤害特效、粒子效果、伤害数字
6. **行动完成**：重置 AV，解除暂停，继续跑条

### 7.2 伤害表现

**伤害数字**：
- 玩家伤害（蓝色），敌人伤害（粉色）
- 显示时间：2秒

**多hit表现**：
- 每hit间隔 200ms
- 每发伤害独立计算防御
- 伤害数字位置随机分布

**轻重区分**（v3.0 设计方向）：
- **普通攻击**：标准伤害数字，少量粒子
- **重击/能量技**：屏幕微震，大号伤害数字，大量粒子，闪屏
- **追加技能**：专属颜色/动画提示，与主技能表现有视觉区分

## 强化系统

### 9.1 强化节奏（v3.0 更新）

- **普通战后**：三选一，标准稀有度池（R 70% / SR 25% / SSR 5%）
- **精英战后**：三选一，**高稀有度池**（R 40% / SR 40% / SSR 20%）
- **可跳过**：玩家可以选择不拿强化

### 9.2 稀有度与数值

- R / SR / SSR 三档
- 数值倍率梯度：**25% / 50% / 100%**

### 9.3 升级规则

- 强化可重复出现并升级（仅**数值类**）
- 机制类强化仅出现一次
- 升级显示：`Lv.1 → Lv.2`
- 升级成长：**+20%/级**

### 9.4 通用强化

- A倍率提升（攻击技能伤害）
- S倍率提升（特技伤害）
- U倍率提升（能量技伤害）
- 攻击力/防御力/速度/生命值成长（固定值）

### 9.5 职业专属强化

**气宗**：
- 内伤倍率提升至 **1.5**
- U触发内伤时附带破防
- 迅击后自身速度提升（百分比）
- 内伤状态下受击有 **35%** 几率暴击，造成 **200%** 伤害

**剑圣**：
- 多段攻击每段伤害提升（每段 +10%）
- 疾风每层速度提升（固定数值）
- 连斩追加概率提升
- U消耗所有连击点释放（最少6），每多消耗1点伤害额外提升 **10%**

**魔导**：
- 盈能每层攻击加成提升
- 盈能每层速度提升（百分比）
- 装填消耗魔力 -1
- 盈能满层时强化爆射伤害提升（百分比）

**判官**：
- 反差系数提升（逆方向额外倍率增加）
- 极值 CD 缩短 1 次行动
- 侵蚀伤害倍率提升
- 阴阳每层被动效果提升（百分比）

## 技术实现

### 12.1 代码结构

- **多文件架构**：入口为 `game-v2/index.html`，逻辑拆分在 `game-v2/js/*.js`，样式在 `game-v2/css/style.css`
- **面向对象设计**：使用 ES6 Class 实现角色和魂灵系统
- **模块化函数**：View（渲染）与 Logic（数据计算）分离

### 12.2 关键类

- `CTBSystem`：时间轴系统
- `Character`：角色基类
- `QiMaster`、`ComboMaster`、`ManaMaster`、`BalanceMaster`：四种角色流派
- `Spirit`：魂灵基类
- `Enemy`：敌人基类 + 各种敌人子类
- `ParticleSystem`：粒子特效系统

### 12.3 v3.0 需新增模块

- **FollowUpSystem**：追加技能触发判定 / UI弹出 / 确认处理
- **RunManager**：单局流程管理（阶段推进、节点类型、胜败判定）
- **RestNode**：修整节点逻辑（商店/事件/回血）
- **SpiritShrine**：魂灵祭坛选择场景

### 12.4 技能/强化架构

- **技能对象化**：`Skill` 以配置 + 实例的形式存在，运行期可叠加强化/修饰
- **修饰器**：`SkillModifier` 提供 `mult/add` 两类伤害修正
- **事件钩子**：`onBeforeAttack` / `onAfterAttack` / `onResourceChange`
- **强化配置表**：通过 `UpgradeConfig` 统一配置

### 12.5 性能优化

- 使用 `requestAnimationFrame` 实现平滑动画
- 时间轴渲染使用 CSS transition
- 粒子系统自动清理，避免内存泄漏
