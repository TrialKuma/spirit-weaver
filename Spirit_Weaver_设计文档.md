## 游戏概述

Spirit Weaver 是一款 Web 端回合制策略肉鸽游戏。玩家选择四种职业之一（气宗/剑圣/魔导/判官），与一位魂灵搭档组成 1+1 双核阵容，通过 13 个节点的单局流程（战斗 + 修整），最终击败 Boss 通关。

**核心战斗**：CTB 时间轴回合制，玩家管理职业专属资源（气/连击/魔力-弹药/平衡），在主动技能循环、追加技能概率触发、魂灵联动三层决策中寻找最优出招。

## 技术实现

### 技术栈

- HTML5 + CSS3 + 原生 JavaScript（无框架、无构建步骤）
- Web Audio API 程序化音效
- WAV 格式职业 BGM（Python 脚本生成）
- 部署：Netlify（`publish: game-v2`）

### 代码结构

入口：`game-v2/index.html`

```
game-v2/js/
├── state.js          # GameState + GameEvents 事件总线
├── utils.js          # CTBSystem, ParticleSystem, Logger
├── sfx.js            # Web Audio API 程序化音效
├── audio.js          # BGM 加载 + UI 音效
├── skills.js         # SkillModifier, createSkill, 伤害计算
├── classes.js        # 4职业类 + Spirit类 + 魂灵目录 + 追加技能定义
├── enemies.js        # Enemy基类 + 11种敌人 + 工厂函数
├── upgrade_config.js # 强化配置表
├── upgrades.js       # 强化应用逻辑
├── run.js            # RunManager (13节点流程)
├── rest.js           # 修整节点 (UI + 事件)
├── ui.js             # UI更新 (HP/资源/技能/Buff/追加)
└── main.js           # 游戏主循环、战斗流程、技能执行、追加执行
```

**脚本加载顺序**（`index.html` 中）：
`state → utils → sfx → audio → skills → classes → enemies → upgrade_config → upgrades → run → rest → ui → main`

### 关键类

| 类 | 职责 |
|----|------|
| `CTBSystem` | 时间轴管理、tick 推进、单位排序 |
| `Character` | 角色基类（HP/Buff/Shield/伤害/统计） |
| `QiMaster` / `ComboMaster` / `ManaMaster` / `BalanceMaster` | 四职业子类 |
| `Spirit` | 魂灵实例，通过 hooks 响应游戏事件 |
| `Enemy` | 敌人基类，子类实现 `planNextAction()` |
| `ParticleSystem` | 粒子特效、伤害数字、技能切入 |
| `RunManager` | 单局流程状态机 |

### 架构模式

- **面向对象**：角色/敌人/魂灵用 ES6 Class
- **事件驱动**：`GameEvents` 总线用于技能/魂灵/强化效果挂接
- **配置驱动**：技能用 `createSkill(config)` 工厂，强化用 `UpgradeConfig` 表
- **Hook 模式**：魂灵通过 `onPlayerAction` / `onBeforeSkill` / `onSpiritTurn` 等钩子响应
- **修饰器模式**：`SkillModifier` 提供技能伤害修正叠加
