# Asset Cut - UI元素抠图 Skill

从游戏截图/UI设计图中识别、裁剪并抠出各个独立元素，输出透明背景的PNG文件。

## 前置条件

需要安装 Python 依赖（全局安装一次即可）：

```bash
pip install Pillow rembg
```

## 使用流程

### 1. 用户提供图片

用户提供一张游戏截图或UI设计图，并说明需要抠出哪些元素。

### 2. 分析图片

使用 `view_file` 工具查看图片，获取图片尺寸：

```python
from PIL import Image
img = Image.open("图片路径")
w, h = img.size
```

目视分析图片中各元素的位置，确定每个元素的裁剪区域坐标（使用比例值 0.0~1.0，适配不同分辨率）。

### 3. 编写配置并执行脚本

使用 `d:\AssetCut\scripts\extract.py` 脚本，创建一个任务配置文件：

```python
# 任务配置示例
SRC_IMG = "源图片路径"
TASK_NAME = "描述性任务名_日期"  # 如 "魂魂食堂_20260306"

# 元素定义：名称 -> (left%, top%, right%, bottom%)
ELEMENTS = {
    "元素名1": (0.10, 0.20, 0.50, 0.60),
    "元素名2": (0.50, 0.30, 0.90, 0.70),
}
```

执行命令：

```bash
python d:\AssetCut\scripts\extract.py --src "图片路径" --task "任务名" --config "配置文件路径"
```

或直接修改 `extract.py` 中的配置后运行：

```bash
python d:\AssetCut\scripts\extract.py
```

### 4. 验证裁剪效果

使用 `view_file` 查看输出的每个元素图片，确认：
- 裁剪位置是否准确（元素是否完整、有无多余内容）
- 去背景效果是否满意

如果位置不对，调整比例坐标后重新执行。

### 5. 输出结构

```
d:\AssetCut\
├── output\<任务名>\    ← 矩形裁剪中间产物（带背景）
├── final\<任务名>\     ← 最终成品（透明背景PNG）
└── scripts\            ← 脚本目录
```

## 坐标估算技巧

- 坐标使用比例值 (0.0~1.0)，格式为 `(left, top, right, bottom)`
- 先从图片视觉上估计元素占图片宽/高的比例
- 第一次裁剪可以稍微放大范围，确认后再收紧
- 重叠元素需要分别裁剪，可能包含其他元素的部分

## 去背景说明

- 使用 **rembg (u2net 模型)** 进行 AI 去背景
- 独立、清晰的元素效果最好（如单独的道具、角色）
- 重叠密集的区域效果可能不理想
- GPU（CUDA）只加速处理速度，不影响质量
- 如果 rembg 不可用，会自动回退为矩形裁剪输出

## 常见元素命名规范

| 类别 | 命名示例 |
|------|---------|
| 角色 | 顾客1_红胡子, 玩家角色, NPC_商人 |
| 道具 | 鱼, 丸子串, 宝剑 |
| UI控件 | 返回按钮, 金币图标, 血条 |
| 场景物件 | 烤炉_左, 桌子, 背景树 |
