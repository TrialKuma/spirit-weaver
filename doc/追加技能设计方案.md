# 追加技能设计方案

**版本**：v3.0
**基于**：设计文档 v3.0 追加技能系统规则

---

## 系统规则回顾

- **触发判定**：触发率绑在**来源技能**上，使用主动技能后做一次概率判定；触发后展示该技能所有符合条件的追加，玩家选一个或跳过
- **触发率范围**：30%-40%
- **资源消耗**：1-2点职业资源（判官为平衡值偏移±1）
- **不连锁**：追加技能不会再触发追加
- **不消耗行动**：不重置AV，属于当前行动的延伸
- **追加分类**：攻击追加 / 功能追加
- **追加与被动**：追加技能的命中不触发职业被动的"叠层"部分（如剑圣疾风叠层、连击积攒），但魔导的盈能被动（攻击后消耗1层回3魔力）正常触发——追加不应与被动脱节

---

## 设计目标

每职业 **2个追加**，每个追加都有明确的"用 vs 不用"决策点。

---

## 气宗 (Qi Master)

**核心张力**：每1点气都在"攒气放崩山"和"花气做别的"之间拉扯。

**来源技能触发率**：

| 来源技能 | 触发率 | 展示的追加 |
|----------|--------|-----------|
| 轻击、迅击 | 33% | 追击 + 劲气（二选一） |
| 其他主动技能 | 33% | 劲气 |

### 追加1：追击（攻击追加）

| 字段 | 值 |
|------|-----|
| ID | `qi_follow_strike` |
| 名称 | 追击 |
| 图标 | 👊 |
| 消耗 | 1 气 |
| 效果 | 追加一次 **100% 攻击力**打击；若目标有【内伤】，伤害 **+50%** |
| 标签 | Light, Melee, FollowUp |

**决策场景**：

| 场景 | 分析 | 倾向 |
|------|------|------|
| 轻击(0气)后触发 | 花1气换100%追击，总输出200%只花1气 | 用 |
| 迅击后触发，目标有内伤已返还2气 | 净消耗1气，追击150%，性价比极高 | 用 |
| 迅击后触发，目标无内伤 | 已花2气，再花1气=3气总输出300% | 看气量 |
| 当前6-7气，省1气下回合刚好够崩山 | 追击会推迟崩山1回合 | 跳过 |

---

### 追加2：劲气（功能追加）

| 字段 | 值 |
|------|-----|
| ID | `qi_follow_force` |
| 名称 | 劲气 |
| 图标 | 💫 |
| 消耗 | 1 气 |
| 效果 | 造成 **60% 攻击力**伤害，推迟敌人行动条 **20%** |
| 标签 | Utility, Melee, FollowUp |

**设计意图**：气宗的核心节奏是"等待→爆发"，劲气用1气换时间——延缓敌人行动意味着自己多攒1-2回气再行动。60%伤害保证打击感，核心价值在行动条控制。

**决策场景**：

| 场景 | 分析 | 倾向 |
|------|------|------|
| 气充裕(8+)，崩山不急 | 1气换敌人延迟，争取更多攒气回合 | 用 |
| 气紧张(4-6)，正在攒崩山 | 有意义的投资，但推迟崩山 | 看敌人意图 |
| 敌人蓄力中，下一击危险 | 推迟敌人行动=延迟威胁 | 用 |
| 气≤2，每点都珍贵 | 花不起 | 跳过 |

## 剑圣 (Combo Master)

**核心张力**：连击点是终结技的弹药，每花1点都在推迟终结技；但剑圣的多动性格鼓励频繁出手。

**来源技能触发率**：

| 来源技能 | 触发率 | 展示的追加 |
|----------|--------|-----------|
| 疾风、连斩 | 35% | 回旋斩 + 破绽（二选一） |
| 其他主动技能 | 35% | 破绽 |

### 追加1：回旋斩（攻击追加）

| 字段 | 值 |
|------|-----|
| ID | `combo_follow_spin` |
| 名称 | 回旋斩 |
| 图标 | ⚔ |
| 消耗 | 1 连击 |
| 效果 | 追加 **80% 攻击力**单段攻击，叠加 **1层疾风** |
| 标签 | Light, Melee, FollowUp |

**打击节奏**：疾风(2hit) + 回旋斩(1hit) = 2+1，有收尾感。

**决策场景**：

| 场景 | 分析 | 倾向 |
|------|------|------|
| 连击0时 | 花不起 | 不可用 |
| 连击1-4时 | 1连击换80%伤害+1疾风，稳定收益 | 用 |
| 连击5-6，接近终结技 | 花1连击推迟终结，但疾风有价值 | 看疾风层数 |
| 疾风已满(10层) | +1疾风溢出浪费，只剩80%伤害 | 看连击量 |

---

### 追加2：破绽（功能追加）

| 字段 | 值 |
|------|-----|
| ID | `combo_follow_expose` |
| 名称 | 破绽 |
| 图标 | 🎯 |
| 消耗 | 1 连击 |
| 效果 | 造成 **50% 攻击力**伤害，敌人防御 **-2**（2回合） |
| 标签 | Melee, Utility, FollowUp |

**设计意图**：DEF-2 在扣减制下对多段攻击收益最大（5段连斩每段多打2=总+10），对终结技（单发）收益低。破绽与回旋斩**同时展示、二选一**——玩家根据后续出招计划选择要伤害+疾风（回旋斩）还是要减防铺路（破绽）。

**触发展示**（疾风/连斩后触发时）：

| 选项 | 消耗 | 效果 | 适合时机 |
|------|------|------|----------|
| 回旋斩 | 1连击 | 80%伤害+1疾风 | 连击少/疾风未满时 |
| 破绽 | 1连击 | 50%伤害+DEF-2(2回合) | 敌人DEF高，接下来要多段攻击时 |
| 跳过 | 0 | 无 | 连击接近终结技阈值时 |

**决策场景**：

| 场景 | 选择 | 理由 |
|------|------|------|
| 疾风未满，需要叠速度 | 回旋斩 | 80%伤害+1疾风 |
| 敌人DEF高，准备连斩 | 破绽 | 每段hit多打2，多段放大 |
| 连击0 | 跳过 | 花不起 |
| 敌人DEF已经0-1 | 回旋斩 | 减防无意义 |
| 疾风已满(10层) | 破绽 | 疾风溢出，减防更有价值 |

## 魔导 (Mana Master)

**核心张力**：弹药是魔导的核心输出载体，追加围绕弹药经济展开。追加触发盈能被动（攻击后消耗1层盈能，回3魔力），不与被动脱节。

**来源技能触发率**：

| 来源技能 | 触发率 | 展示的追加 |
|----------|--------|-----------|
| 强化射击、强化爆射 | 35% | 余波 |
| 装填 | 40% | 过载充能 |

### 追加1：余波（攻击追加）

| 字段 | 值 |
|------|-----|
| ID | `mana_follow_aftershock` |
| 名称 | 余波 |
| 图标 | 🔫 |
| 消耗 | 1 弹药 |
| 效果 | 追加远程射击（倍率待平衡），触发盈能被动（消耗1层盈能，回3魔力） |
| 标签 | Ranged, FollowUp |

**设计意图**：消耗弹药而非魔力，与魔导"弹药=输出载体"的逻辑统一。触发盈能被动意味着余波不仅是伤害，还是资源回收——多打一发子弹=多回一次魔力。伤害倍率待全局平衡后确定。

**盈能交互**（每次攻击消耗1层盈能回3魔力，层数逐发递减）：

```
示例：5层盈能，强化射击 + 余波
  强化射击：5层加成 → 攻击 → 消耗1层 → 4层，+3魔力
  余波追加：4层加成 → 攻击 → 消耗1层 → 3层，+3魔力
  总计：消耗2弹药，消耗2层盈能，回6魔力
```

**决策场景**：

| 场景 | 分析 | 倾向 |
|------|------|------|
| 弹药充裕(3+) | 多打一发+回魔，稳赚 | 用 |
| 弹药剩2，强化射击后剩1 | 再花1发只剩0，进入无弹药模式 | 看局势 |
| 盈能层数高 | 额外消耗1层盈能=多回3魔力 | 用 |
| 盈能0层 | 不回魔，纯消耗弹药 | 看弹药量 |

---

### 追加2：过载充能（功能追加）

| 字段 | 值 |
|------|-----|
| ID | `mana_follow_overcharge` |
| 名称 | 过载充能 |
| 图标 | ⚡ |
| 消耗 | 4 魔力 |
| 效果 | 额外装填 **1发弹药**，消耗4魔力触发盈能（+2层盈能） |
| 标签 | Reload, Utility, FollowUp |

**设计意图**：叫"过载"就要有 all-in 的感觉。装填消耗6魔力，过载再花4魔力——加起来是10魔力全部清空。代价是魔力归零，收益是多1发弹药+2层盈能。40%触发率意味着玩家会**赌**：这次存满10魔力，是不是能触发过载？

**资源流分析**：

```
普通装填：       6魔力 → 3弹药 + 3层盈能
装填+过载充能：  10魔力 → 4弹药 + 5层盈能(满)，魔力归零

4发弹药 × 每发消耗1盈能回3魔 = 最多回12魔力（但盈能逐发递减）
实际：5层→4→3→2→1→0，4发弹药消耗4层，回12魔力，剩1层盈能
```

**决策场景**：

| 场景 | 分析 | 倾向 |
|------|------|------|
| 魔力满(10)，装填后剩4 | All-in！10→0换4弹药+5盈能，后续射击逐步回魔 | 用 |
| 魔力不够10，装填后剩<4 | 花不起 | 不可用 |
| 盈能已经4-5层 | 过载叠盈能可能溢出 | 看弹药需求 |
| 敌人血量低，需速战速决 | 多1发弹药可能就是击杀 | 用 |

**赌博心理**：玩家知道过载有40%触发，可能会刻意攒满10魔力再装填，"赌"过载触发。赌中了 = all-in 爆发；没中 = 装填后剩4魔力，也不亏。这种"是否为触发机会做准备"的前置决策是追加系统最有趣的副产品。

## 判官 (Balance Master)

**不设追加技能。**

**理由**：判官在核心重设计（v3.0 极值决策型）后，基础循环本身决策密度已经足够高：
- **每回合选择**：顺方向推条 vs 逆方向兑现反差（|balance| × 15% 额外倍率）
- **极值三选一**：同方向增强 / 逆方向反差满额 / 宣判归零爆发
- **反差机制**已替代了原本"交叉方向追加"的设计目标

追加技能的核心价值是"给线性循环增加分叉"，判官的循环本身就是分叉，再加追加会造成信息过载。

## 设计原则

1. **每个追加都有"不用"的理由** —— 气宗攒气、剑圣攒连击、魔导省弹药、判官保持推进动量
2. **追加与被动不脱节** —— 魔导余波正常触发盈能被动（消耗1层回3魔），不做例外
3. **消耗与职业核心资源一致** —— 魔导攻击追加消耗弹药（输出载体统一），过载消耗魔力（装填环节资源）
4. **追加伤害倍率偏低但不为零** —— 功能追加也带伤害，保证打击反馈感
5. **判官不设追加** —— 核心重设计后决策密度已足够（反差机制+极值三选一），追加会信息过载
6. **过载类追加有真实代价** —— 4魔力 all-in 清空资源，40%触发率制造"赌不赌"的前置决策
7. **同源追加同时展示** —— 同一技能可触发的多个追加同时展示（回旋斩vs破绽、追击vs劲气），玩家选一个或跳过
8. **倍率待全局平衡** —— 追加技能的具体伤害倍率在全部职业技能定型后统一调整

## 待讨论

- **余波伤害倍率**：需要在全局平衡中确定，避免 2弹药总输出过于接近3弹药
- **追加UI**：计划与底部技能栏合并（平时灰色，触发时高亮），取代当前的弹窗式设计
- **追加触发展示**：已确认触发率绑在来源技能上，触发后展示所有符合条件的追加，玩家选一个或跳过
- **被动交互规则**：需要统一"追加是否触发被动"的规则，目前魔导盈能需要触发但剑圣疾风/连击不触发，存在不一致
- **判官核心重设计**：已完成（v3.0 极值决策型），确认不设追加技能
