# 设计讨论记录：Spirit Weaver 方向梳理与重设计

**日期**：2026.4.13-4.24
**参与**：用户 + 师爷

## 一、起因：对项目的重新审视

### 核心疑问
AI降低实现门槛后，重新审视项目——"这套东西底子到底好不好玩？"
作为成熟项目推出的话，目前设计让人感觉不够好玩。

### 设计演化时间线
1. **2016-2017**：原始ARPG设计（`Main.xlsx`）——15+职业共享4种施法资源（气息/连击/元素/信念），斗环被动树（每职业8-10个被动，含互斥分支），3D不锁定战斗
2. **中间阶段**：TRPG化设计（`技能设计示例-雷克老板.xlsx`、`战斗规则Ver1.0.xlsx`）——4行动点/回合，追加技能链，同一角色适配4种资源的完整示例
3. **收敛至v1**：回合肉鸽（Spirit Weaver CTB版）——CTB时间轴，4固定职业各绑1种资源，每职业3个主动技能
4. **新架构（当前）**：角色底盘+魂灵定义build → 2×2验证后发现回合制战斗过程不好玩 → **转向即时攻防制**

## 二、第一轮设计：新架构（角色底盘+魂灵定义build）

### 核心变更
- 角色是"底盘"（技能+资源系统），魂灵提供被动和build方向
- 角色原有被动能力移至魂灵系统
- 角色的锤子改变技能形态，魂灵的锤子改变被动效果和魂灵攻击

### 已完成的设计
- **气宗**：4基础技能（轻拳/迅击/崩山/架势）+ 12个锤子变体。详见 `新架构-气宗与气息魂设计.md`
- **剑圣**：4基础技能（快斩/连斩/终结技/见切）+ 含乱舞序破急追加链。详见 `新架构-剑圣设计草案.md`
- **气息魂（被动型）**：内伤系统 + 旋风斩/续剑攻击技能 + 三条build分支（内爆/重伤/回流）
- **连击魂（手操型）**：疾风系统 + 斩风/飞驰步/交错之斩 + 三条分支（乘风/裂刃/狂风）

### 两类魂灵对比
| | 气息魂 | 连击魂 |
|---|---|---|
| **类型** | 被动型（不走条，自动响应） | 手操型（走条，玩家选技能） |
| **核心机制** | 【内伤】debuff目标 | 【疾风】buff自身速度 |
| **鼓励节奏** | 攒资源→大额消耗(≥6)→触发 | 高频出手→叠疾风→选时机花 |

### 跨职业兼容性设计
- 魂灵触发条件统一为"资源变动"
- 判官：按翻转时触发
- 魔导：只算魔力消耗，弹药不计入

## 三、2×2验证结果与转向

### 验证结论
2个职业 × 2个魂灵实装后验证——**构筑层做厚了也救不了战斗过程的体感。**

核心问题不是内容量、不是魂灵差异、不是build深度。是**"看条等轮到你→从按钮里选一个→看数字跳出来"这个过程本身不好玩**。回合制的moment-to-moment体验下限就在那里。

### 决定：转向即时攻防制

**目标体验**：
- 没有意图系统，玩家需要即时反应
- 怪物按AI出招，有动作前摇（不是意图图标，是看动画/预兆判断）
- 核心循环是"贪输出 vs 防御"的即时攻防
- 类Punch-Out的攻防节奏——不需要走位，面对面出招时机博弈

**评估结论**：
- HTML/JS+Canvas可以做，不需要换引擎
- 不需要走位/碰撞检测/物理引擎，两个单位面对面，判定只靠时间窗口
- AV系数不做，速度比维持3:1
- 全局CTB时间轴可能改为技能独立CD制（远期考虑，当前先验证即时攻防手感）

### 可复用的设计资产
- 4种资源系统（改成实时回复/实时攒）
- 技能池+锤子变体系统
- 魂灵被动触发条件不变
- 见切的完美格挡在即时制下终于有真正意义
- 疾风叠层=攻击速度/CD缩短

### 需要废弃/重做的
- CTB行动条系统
- 意图系统
- 回合概念
- 敌人AI从"轮到我出招"变成"实时决策+前摇动画"

## 四、已实现的即时制原型进展

### 敌人序列帧动画系统
- AI生成12帧敌人序列帧（暗紫色恶魔）
- 去白底 + 内容检测切图 + 脚底对齐
- `EnemySpriteAnim` 帧动画系统：支持前摇/出招/受击/恢复序列
- 帧分配：待机(0-2) / 攻击前摇(2-3) / 攻击出招(5-6) / 受击(8) / 恢复(10-11)

### 攻击时序改造
旧流程：执行伤害 → 播cut-in → 等待 → 播特效
新流程（已实现）：
1. **t=0**：前摇帧循环 + 喊话同时弹出
2. **t≈900ms**：前摇结束 → 执行伤害数据 → 出招帧+冲刺同时启动
3. **t≈1030ms**：冲到位 → 命中特效+伤害数字+屏幕震动+玩家受击
4. **t≈1430ms**：回待机帧
5. **t≈2030ms**：结束回合

### UI布局调整
- 敌人：去掉属性显示，血条变头顶6px窄条无数值，enemy-box放大200×240透明底
- 玩家：血条移到角色框顶部，属性(ATK/DEF/SPD)并到左下角资源面板
- 敌人图片水平翻转面朝玩家
- 攻击冲刺改为JS直接操控transform（避免CSS animation冲突）

## 六、设计原则（从讨论中沉淀）

1. **角色是底盘，魂灵定义build**——角色提供操作手感差异，魂灵提供策略方向差异
2. **一个行动，双重含义**——玩家的每个操作同时在驱动战斗效果和影响魂灵状态
3. **不做全局AV系数**——时间管理通过具体技能推条效果实现（震破、飞驰步等）
4. **速度比维持3:1**——玩家应该能规避大部分伤害，但贪输出有风险
5. **魂灵被动触发条件**：单次消耗≥6资源触发（非累计）；旋风斩用累计12独立计数
6. **回流改版**：内伤被消耗时触发（不管谁消耗），解决"魂灵先吃"的时序问题
7. **多段技能倍率补偿约+25%**——补偿减法防御下每段各自扣DEF的损失
8. **见切完美格挡**：基于行动值窗口的时机判定，奖励伤害而非奖励生存（保底护盾+反击80%，完美→反击200%+2连击）
