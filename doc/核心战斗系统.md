# 核心战斗系统

**基于**：设计文档 v3.0

---

## CTB 时间轴系统 (Conditional Turn-Based)

**核心公式**：
- **行动值 (AV)**：`AV = 10000 / Speed`
- **跑条机制**：所有单位在同一条轨道上跑条，AV 归零者获得行动权
- **行动消耗**：标准行动后，重置 `AV = 10000 / Speed`
- **暂停机制**：当有单位到达行动条底部（AV归零）时，所有单位的AV停止减少，直到该单位行动完成

**动态跑条速度**（已实现）：
- 当下一个行动单位与当前单位**同属一方**时（连续玩家回合），跑条动画加速至 **5 倍速**
- 当下一个行动单位为**对方**时（玩家→敌人交替），恢复正常速度，自然制造节奏停顿
- **目的**：消除连续行动之间的等待感，同时保留速度差带来的"多动"体验

**实现细节**：
- 每 tick 所有单位 AV -1
- 当 AV ≤ 0 时，该单位获得行动权
- 行动完成后重置 AV，解除暂停状态
- 敌人攻击时也会暂停行动条，等待表现播放完成
- 常量：`CTB_NORMAL_SPEED = 0.5`，`CTB_FAST_SPEED = 2.5`

## 伤害计算系统

**基础公式**：
```
baseDamage = buffedAtk × baseMultiplier
buffedAtk = baseAtk × (1 + buff%) × (1 + 职业被动%)
最终伤害 = max(1, baseDamage × modifiers — enemyDef)
```

- 多hit技能：每发独立计算防御
- 防御减伤是直接减法（非百分比），最低伤害为 1

## Buff 系统

**类型**：
- **层数制**：如【蓄力】（层数消耗）
- **回合制**：如【易伤】（持续回合，每回合结束 duration-1）
- **极性制**：如【平衡状态】（正负值影响攻防）
- **永久制**：`duration: -1` 的 Buff 不会自动消失

**Buff 处理时机**：
- **回合开始**：DOT 伤害、HOT 回复
- **回合结束**：持续时间递减，到期移除
- **状态变化时**：`recalculateStats()` 重算速度/攻击/防御倍率

---

## 战斗回合流程

1. **行动条跑条**：所有单位 AV 持续减少（同侧连续时动画加速）
2. **单位就绪**：当单位 AV ≤ 0 时，暂停所有单位，该单位获得行动权
3. **执行行动**：
   - 玩家：选择技能使用
   - 敌人：执行意图行动
4. **追加判定**：
   - 概率检查是否触发追加技能
   - 若触发：技能面板底部追加技能高亮，玩家选择使用或跳过
   - 若未触发：直接进入下一步
5. **表现播放**：伤害特效、粒子效果、伤害数字
6. **行动完成**：重置 AV，解除暂停，继续跑条

---

## 伤害表现

**伤害数字**：
- 玩家伤害（蓝色），敌人伤害（粉色）

**多hit表现**：
- 每hit间隔 200ms
- 每发伤害独立计算防御
- 伤害数字位置随机分布

**轻重区分**（已实现）：
- **普通攻击**：标准伤害数字，少量粒子
- **重击/能量技**：屏幕微震，大号伤害数字，大量粒子，闪屏
- **追加技能**：金黄色数字/特效（`#ffd54f`），与主技能有视觉区分

## 意图系统

敌人在行动前展示下一步意图，为玩家提供可应对的信息。

**已有意图类型**：
- `attack`：攻击（显示预计伤害）
- `defend`：防御（增加临时防御 buff）
- `buff`：自我强化（护盾、加速、加攻）
- `debuff`：对玩家施加减益（减速、减攻）
- `charge`：蓄力（可被能量技打断）
- `stun`：眩晕（被打断后无法行动）
